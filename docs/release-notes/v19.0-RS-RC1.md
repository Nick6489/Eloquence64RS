# Eloquence64RS 19.0-RS-RC1

This is the first release candidate under the Eloquence64RS name. It is an
unofficial, community-maintained continuation of Eloquence64 and is intended
to replace an existing Eloquence64 installation.

## Highlights

- Replaces the Python-based 32-bit helper with a statically linked Rust host.
- Uses authenticated, versioned binary communication over private child-process
  pipes; no listening socket or helper Python installation is required.
- Preserves NVDA speech indexes and completion notifications while streaming
  audio from the legacy 32-bit Eloquence engine to 64-bit NVDA.
- Fixes Say All stalls after phrase-final em dashes.
- Fixes audible speech-index markers.
- Fixes the first attempt to switch back to Eloquence failing after selecting a
  different synthesizer.
- Includes Traditional-to-Simplified Chinese preprocessing for the Mandarin
  voice and retains the existing Eloquence language and dictionary support.

## Installation

Remove or disable the IBMTTS add-on before installing. Eloquence64RS replaces
Eloquence64 and should not be installed alongside another Eloquence add-on.

After updating, users who need Eloquence on logon or other secure screens must
open **NVDA Settings > Eloquence** and run **Copy Helper to System Config (for
Logon Screen)** again.

## Release-candidate status

This build has passed the automated test suite and extended interactive NVDA
use, but the native host is a substantial architectural change. Please report
speech stalls, synthesizer switching failures, audio-device issues, or secure
screen failures with an NVDA debug log and reproduction steps.
