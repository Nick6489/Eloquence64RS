# Eloquence64RS 19.0-RS-RC3

This release candidate improves Say All reliability, modernizes the text-to-engine
boundary, and makes native-host updates safer to test on secure screens.

## Changes since RC2

- Moves Eloquence Text construction into a pure, independently tested builder
  that produces voice-appropriate engine bytes before the Rust transport layer.
- Removes mutable transport-layer language encoding state so language changes
  cannot select the wrong encoding for queued text.
- Corrects `BreakCommand` pause durations at speech rates exactly 43, 60, and 75.
- Preserves NVDA's chunk separator before full month names, avoiding speech
  concatenation introduced by the phrase-final em-dash workaround.
- Statically links the native host C runtime for more reliable deployment.
- Warns when the helper copied into NVDA's `systemConfig` differs from the helper
  shipped with the installed add-on.
- Adds regression coverage for text encoding, pause policy, language scoping,
  native-host protocol behavior, Say All progress, and secure-screen helper
  detection.

## Installation

Remove or disable the IBMTTS add-on before installing. Eloquence64RS replaces
Eloquence64 and should not be installed alongside another Eloquence add-on.

After updating, users who need Eloquence on logon or other secure screens must
open **NVDA Settings > Eloquence** and run **Copy Helper to System Config (for
Logon Screen)** again. RC3 will warn after startup if the copied helper is out
of date.

## Release-candidate status

Please test normal reading, Say All in browsers and documents, automatic
language switching, punctuation pauses, synthesizer switching, cancellation,
secure screens, and different audio devices. Report stalls, missing or repeated
speech, clicks, timing problems, or crashes with an NVDA debug log and clear
reproduction steps.
